# クレクレコイン (KC) × Map Album 統合計画

本ドキュメントでは、仮想通貨「クレクレコイン (KC)」を「Map Album」内で活用するための機能定義と、技術的な連携方法について記述します。

---

## 1. KC の活用機能 (Features)

Map Album の「位置情報」と「写真共有」を活かし、以下の3つの機能を実装します。

### A. 新規地域開拓ボーナス (Region Discovery Bonus)
- **概要**: ユーザーがこれまでに写真を撮ったことがない「新しい市や郡（海外ではそれに相当する行政区画）」で写真を撮影・投稿した際に、ボーナスとして **10,000 KC** が付与されます。
- **送金元**: SYSTEM (GENESIS アカウント)
- **判定**: 位置情報（緯度・経度）から行政区画を逆ジオコーディングで取得し、そのユーザーの投稿履歴と照合して判定します。

### B. シークレット写真機能 (Secret Photo)
- **概要**: 投稿者が「シークレット」として設定した写真は、閲覧者が **100 KC** を支払うまでモザイク処理やプレビュー画像で表示されます。
- **送金フロー**: 閲覧者（Viewer） → 投稿者（Owner）へ直接 100 KC を送金。
- **解放条件**: 送金が完了（取引IDが確認）された時点で、フル解像度の写真が閲覧可能になります。

### C. 投げ銭機能 (Tipping)
- **概要**: 良いと思った写真に対して、閲覧者が自由な金額を設定して KC を送ることができる機能です。
- **送金フロー**: 閲覧者（Viewer） → 投稿者（Owner）へ直接送金。
- **金額設定**: 支払う側が 1 KC 以上の任意の額を入力して実行します。

## 2. システム構成・同期方法 (Technical Integration)

KC の署名検証ベースの仕組みを活かし、Map Album 側で秘密鍵を管理しない安全な構成をとります。

### A. ユーザーアカウントの紐付け
Map Album の `users` テーブルに、KC ウォレットのアドレスを登録するカラムを追加します。
- `kc_address`: ユーザーの公開鍵から導出されたアドレス。

### B. 送金処理の仕組み (API連携)
送金は Map Album のフロントエンドから KC サーバーの API を呼び出すことで実行します。

1.  **ユーザー操作**: 「100 KC 払う」または「投げ銭」ボタンをクリック。
2.  **署名**: ユーザーが（ブラウザに保存された秘密鍵を用いて）送金データに署名を作成。
3.  **API呼び出し**: Map Album フロントエンドから KC サーバーの `/api/send` へリクエスト。
4.  **反映**: 送金成功後、取引 ID (txId) を Map Album 側に記録し、写真の閲覧権限などを更新。

## 3. 実装上の注意
- **エフェクト・通知**: 演出用のエフェクト（チャリンという音など）や、独自の通知機能は実装対象外とします。
- **自動報酬**: 複雑なバックエンドのバックグラウンド処理は避け、写真投稿時の API レスポンスやシンプルなトリガーをベースに実装します。
